6 ИЮЛЯ 2015 Г.
Хранить файлы в GridFS в приложении Meteor
Задний план

Недавно я начал новый проект на основе Meteor, который включает в себя возможность для пользователей сохранять свои собственные изображения в облаке. Я рассмотрел несколько вариантов для этого, от хранения до локальной FS до Amazon S3. В конце концов я решил, что было бы удобно хранить файлы в GridFS Mongo. Мне все равно нужен кластер Mongo, так почему бы и не хранить там файлы. 
Но оказалось, что в Meteor для GridFS нет встроенной поддержки.

Существует множество плагинов, из которых наибольшая из них - сбор файлов и коллекция-fs . Я решил не использовать ни одного из них, потому что они вроде бы слишком много делают для меня, например, добавляют функции загрузки HTTP. CollectionFS - это большой набор плагинов на самом деле, который пытается многое сделать (но, к примеру, не очень хорошо сочетается с последним плагином Meteor и плагином проверки аудита). 
Итак, через некоторое время я наткнулся на этот ответ StackOverflow , который дал мне представление о том, как быстро реализовать хранилище на базе GridFS.

Meteor Hack

Я использую следующий код в Meteor 1.1.0.2, не все здесь является частью публичного API, поэтому он может не работать для других версий. 
Чтобы все было просто, я решил просто использовать драйвер Node.js от Mongo, который используется внутри самого Meteor.





// Access GridStore API
var GridStore = MongoInternals.NpmModule.GridStore;

// Access default mongo database
var db = MongoInternals.defaultRemoteCollectionDriver().mongo.db;

// Reading file from the GridFS by filename
new GridStore(db, filename, 'r').open(Meteor.bindEnvironment(function(err, gridStore) {
    var stream = gridStore.stream();
    // .. do something with stream
}));

// Writing data to GridFS
new GridStore(db, filename, 'w').open(Meteor.bindEnvironment(function(err, gridStore) {
    // Write string or buffer data to file
    gridStore.write(data, true, function(err, doc) { /* ... */ });

    // Or write local FS file
    gridStore.writeFile(localname, function(err, doc) { /* ... */ });
}));




Странно, что GridStore API не предоставляет способ получить файл с идентификатором, но только с именем файла, которое может быть не уникальным. Поэтому вы должны заботиться о уникальности имени файла (возможно, используя некоторые комбинации ObjectID).
